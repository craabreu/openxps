LockstepIntegrator¶

class openxps.LockstepIntegrator(physical_integrator, extension_integrator=None, assume_kick_first=False)[source]¶

An integrator that advances the physical and extension systems in lockstep.

This class integrates both the physical and extension systems simultaneously, then synchronizes the contexts after each step. It assumes that both integrators apply all forces at the beginning of each time step (a kick-first scheme). If either integrator does not follow this scheme, the overall integration will be incorrect.

The step sizes of the physical and extension integrators must be equal.

Parameters:

  • physical_integrator (Integrator) – The integrator for the physical system.

  • extension_integrator (Integrator | None) – The integrator for the extension system. If None, a copy of the physical integrator is used.

  • assume_kick_first (bool) – If True, skip the validation that checks whether the integrators are known to follow a kick-first scheme. Use this at your own risk if you know your integrators are compatible. Default is False.

Raises:

ValueError – If the physical and extension integrators do not follow a kick-first scheme or do not have the same step size.
