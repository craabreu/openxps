ForceMatchingRegressor¶

class openxps.ForceMatchingRegressor(dynamical_variables, num_kernels, *, validation_fraction=0.1, batch_size=256, num_epochs=50, patience=10, learning_rate=0.002, weight_decay=0.0001, accelerator='auto', num_workers=0)[source]¶

Potential regressor from sampled position/force pairs.

A potential function in a \(d\)-dimensional variable space is approximated as:

\[U({\bf s}) = \sum_{m=1}^n w_m \exp\left( -\frac{1}{2} \sum_{k=1}^d \frac{\delta_k^2(s_k - c_{m,k})}{\sigma_{m,k}^2} \right)\]

where the \(n\) weights \(w_m\), kernel bandwidths \(\sigma_m\), and kernel centers \({\bf c}_m\) are adjustable parameters. The displacement function \(\delta_k(x)\) depends on the periodicity of the variable \(s_k\) and is defined as:

\[\begin{split}\delta_k(x) = \left\{\begin{array}{ll} x & \text{if }s_k\text{ is non-periodic} \\ \frac{L_k}{\pi}\sin(\frac{\pi x}{L_k}) & \text{if }s_k\text{ has period }L_k \end{array}\right.\end{split}\]

Given \(N\) position-force pairs \(({\bf s}_i, {\bf F}_i)\), the parameters are adjusted by minimizing the mean squared error loss:

\[L = \frac{1}{N} \sum_{i=1}^N \left\| {\bf f}({\bf s}_i) - {\bf F}_i \right\|^2,\]

where \({\bf f}({\bf s}) = -\nabla_{\bf s} U({\bf s})\) is the predicted force.

Parameters:

  • dynamical_variables (Sequence[DynamicalVariable]) – The dynamical variables to be used in the potential.

  • num_kernels (int) – The number of kernels to be used in the potential.

Keyword Arguments:

  • validation_fraction – The fraction of the data to be used for validation.

  • batch_size – The batch size to be used for training.

  • num_epochs – The number of epochs to be used for training.

  • patience – The patience for early stopping.

  • learning_rate – The learning rate to be used for training.

  • weight_decay – The weight decay to be used for training.

  • accelerator – The accelerator to be used for training. Valid options can be found here.

  • num_workers – The number of data loading workers to be used for training.
